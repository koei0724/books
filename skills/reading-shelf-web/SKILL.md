# 독서 기록 서재 웹

## 개요

이 스킬은 가벼운 독서 기록 서재 웹앱을 만들고 유지보수할 때 사용한다. 기본 제품 형태는 한국어 UI이며, 사용자는 책을 검색하고, 책 표지를 서재에 추가하고, 표지를 열어 상세 정보를 확인한 뒤 읽은 날짜와 독후감을 저장할 수 있어야 한다.

## 작업 흐름

1. 수정하기 전에 현재 프로젝트 구조를 먼저 확인한다. 프레임워크가 없다면 `index.html`, `styles.css`, `app.js` 기반의 정적 구현을 우선한다.
2. 핵심 사용자 흐름을 유지한다: 책 검색, 검색 결과 클릭으로 서재에 추가, 서재 표지 클릭으로 상세 열기, 읽은 날짜와 독후감 편집, 저장, 삭제, 새로고침 후 기록 유지.
3. 사용자가 다른 API를 지정하지 않았다면 브라우저에서 직접 Open Library를 사용해 책 검색과 표지 이미지를 가져온다. 검색, 표지, 저장 동작을 바꾸기 전에는 `references/open-library-vercel.md`를 읽는다.
4. 개인 독서 기록은 기본적으로 `localStorage`에 저장한다. 기존 앱을 수정할 때는 저장 스키마를 안정적으로 유지하고, 이미 `reading-shelf.books.v1` 키가 있다면 특히 보존한다.
5. UI는 마케팅 랜딩 페이지가 아니라 실제로 쓸 수 있는 서재 경험에 집중한다. 한국어 문구, 반응형 레이아웃, 가능한 경우 실제 표지 이미지, 표지가 없을 때도 완성도 있는 대체 표지를 사용한다.
6. 문서 작업과 내부 개발 검증을 먼저 끝낸다. 대표 흐름을 검증한다: 검색 결과 표시, 추가 시 서재 항목 1개 생성, 중복 추가 시 기존 항목 열기, 저장한 날짜/독후감이 새로고침 후 유지, 삭제 시 제거, 모바일 폭에서 가로 넘침 없음.
7. 사용자가 명시적으로 "배포해줘", "Vercel에 올려줘", "프로덕션 배포해줘"처럼 배포를 요청하지 않았다면 배포 명령을 실행하지 않는다. 검증 결과와 배포 준비 상태만 보고하고 멈춘다.

## 구현 메모

- 검색 결과는 외부 API 데이터를 내부 책 레코드로 정규화한다. 필드는 `id`, `title`, `authors`, `firstPublishYear`, `coverId`, `isbn`, `addedAt`, `readDate`, `review`를 사용한다.
- 검색 결과와 서재 항목 클릭은 이벤트 위임을 우선한다. 다시 렌더링할 때마다 많은 리스너를 다시 묶지 않기 위해서다.
- 외부 표지 이미지가 없으면 텍스트와 CSS로 대체 표지를 렌더링한다. 빈 이미지 박스를 남기지 않는다.
- 사용자에게 보이는 책 데이터와 독후감 텍스트는 HTML 문자열 보간이 아니라 `textContent`로 넣어 안전하게 렌더링한다.
- 백엔드가 없어도 앱이 유용해야 한다. 사용자가 기기 간 동기화를 요청하면 별도의 백엔드/데이터베이스 기능으로 다룬다.

## 배포

배포는 별도 게이트다. 구현, 문서 작업, 내부 개발 검증이 끝났더라도 사용자가 명시적으로 배포를 요청하지 않으면 Vercel CLI나 다른 배포 명령을 실행하지 않는다. 이 경우 최종 응답에는 검증 완료 상태와 "배포 요청 시 배포 가능"하다는 점만 짧게 보고한다.

사용자가 배포를 요청한 경우에만 Vercel 배포를 진행한다. 먼저 사용자 맥락에서 대상 계정 또는 scope를 확인한다. `.vercel/project.json`이 있으면 기존 연결을 사용하고, 없으면 Vercel CLI로 프로젝트를 만들거나 연결한다. 프레임워크나 출력 디렉터리를 새로 도입하지 않았다면 프로젝트 루트의 정적 파일을 배포한다.

배포 전에는 Playwright 로그, 스크린샷 같은 로컬 검증 산출물이 업로드되지 않도록 제외한다.

배포 후에는 프로덕션 URL을 열어 페이지 로드와 책 검색 1회를 최소한으로 검증한다.
